# 更新日志

本文件记录项目的所有重要变更。

## [1.0.1] - 2026-03-21

### 变更

- **扩展源文件优先级工作流**：新增用户预处理文档作为 PDF 回退前的次要来源
  - 源文件优先级现为：TeX 源文件 → 用户预处理文档（`.md`、`.json`、图片）→ PDF 提取
  - 用户可提供自行提取的内容（如之前解析 PDF 的结果），技能将在回退到原始 PDF 前先读取这些文件

### 新增

- **日志与数据持久化**：新增运行日志和输出数据保存要求
  - 日志必须保存到 `logs/` 目录下的时间戳文件中
  - 输出数据必须以结构化格式（JSON、CSV 等）保存到 `outputs/` 或 `results/` 目录
  - 配置快照必须与输出一同保存，以确保可复现性

- **结果验证流程**：强制要求将复现结果与论文报告指标进行对比
  - 从论文源文件中提取定量指标
  - 从复现输出中计算相同指标
  - 并排记录论文值与复现值
  - 当结果偏差较大时，调查原因并修复问题
  - 根据论文领域定义可接受的误差范围

### 更新

- `SKILL.md`：新增第 6 节（日志与数据持久化）和第 7 节（结果验证与对比）；重新编号后续章节
- `README.md` 和 `README_zh-CN.md`：更新功能描述，增加日志和验证要求

## [1.0.0] - 2026-03-11

### 新增

- 初始化 Paper Repro Python 技能项目，包含核心文件：

  - 双语 README 文件（`README.md`、`README_zh-CN.md`）
  - Codex 代理配置（`agents/codex.yaml`）
  - 技能行为规范（`SKILL.md`）

### 功能

- **TeX 优先工作流**：优先读取本地 TeX 源文件（`.tex`、`.bib`、样式文件、图片），而非 PDF 提取
- **PDF 回退机制**：当 TeX 不可用时，提供完整保真的 PDF 转 Markdown 提取
- **图片粒度控制**：默认一图一文件；仅在需要对比时使用多子图合并
- **多平台兼容**：支持 Codex、Claude Code 和 OpenClaw
- **双语文档**：自动维护中英文 README 文件
- **隐式技能调用**：为 Codex 添加 `allow_implicit_invocation` 策略
